DLL Side-Loading

Programs may specify DLLs that are loaded at runtime. Programs that improperly or vaguely specify a required DLL may be open to a vulnerability in which an unintended DLL is loaded. Side-loading vulnerabilities specifically occur when Windows Side-by-Side (WinSxS) manifests [1] are not explicit enough about characteristics of the DLL to be loaded. Adversaries may take advantage of a legitimate program that is vulnerable to side-loading to load a malicious DLL. [2]

Adversaries likely use this technique as a means of masking actions they perform under a legitimate, trusted system or software process.

ID: T1073
Tactic: Defense Evasion
Platform: Windows
Data Sources: Process use of network, Process monitoring, Loaded DLLs
Defense Bypassed: Process whitelisting, Anti-virus
CAPEC ID: CAPEC-641
Version: 1.0

APT32 ran legitimately-signed executables from Symantec and McAfee which load a malicious DLL. The group also side-loads its backdoor by dropping a library and a legitimate, signed executable (AcroTranscoder).[19][27][28]

BADNEWS typically loads its DLL file into a legitimate signed Java or VMware executable.[3][4]

DLL side-loading has been used to execute BBSRAT through a legitimate Citrix executable, ssonsvr.exe. The Citrix executable was dropped along with BBSRAT by the dropper.[9]

Denis exploits a security vulnerability to load a fake DLL and execute its code.[19]

FinFisher uses DLL side-loading to load malicious programs.[6][7]

HTTPBrowser has used DLL side-loading.[11]

HyperBro has used a legitimate application to sideload a DLL to decrypt, decompress, and run a payload.[20]

menuPass has used DLL side-loading to launch versions of Mimikatz and PwDump6 as well as UPPERCUT.[12][25]
OwaAuth

OwaAuth has been loaded onto Exchange servers and disguised as an ISAPI filter (DLL file). The IIS w3wp.exe process then loads the malicious DLL.[11]

A Patchwork .dll that contains BADNEWS is loaded and executed using DLL side-loading.[21]
PlugX

PlugX has used DLL side-loading to evade anti-virus.[10][11][2][12][13]

Sakula uses DLL side-loading, typically using a digitally signed sample of Kaspersky Anti-Virus (AV) 6.0 for Windows Workstations or McAfee's Outlook Scan About Box to load malicious DLL files.[8]

During the T9000 installation process, it drops a copy of the legitimate Microsoft binary igfxtray.exe. The executable contains a side-loading weakness which is used to load a portion of the malware.[16]
Threat Group-3390

Threat Group-3390 has used DLL side-loading, including by using legitimate Kaspersky antivirus variants in which the DLL acts as a stub loader that loads and executes the shell code.[11][22][23][20]

Tropic Trooper has been known to side-load DLLs using a valid version of Windows Address Book executable with one of their tools.[29]
Wingbird

Wingbird side loads a malicious file, sspisrv.dll, in part of a spoofed lssas.exe service.[14][15]
ZeroT

ZeroT has used DLL side-loading to load malicious payloads.[17][18]

Mitigations

Mitigation Description
Audit

Use the program sxstrace.exe that is included with Windows along with manual inspection to check manifest files for side-loading vulnerabilities in software.
Restrict File and Directory Permissions

Install software in write-protected locations.
Update Software

Update software regularly to include patches that fix DLL side-loading vulnerabilities.

Detection

Monitor processes for unusual activity (e.g., a process that does not use the network begins to do so). Track DLL metadata, such as a hash, and compare DLLs that are loaded at process execution time against previous executions to detect differences that do not correlate with patching or updates.
